## 1. コンセプト

1年を1周で表現するアナログ時計型計器。
針の位置で「今年のどのあたりにいるか」を直感的に把握できるツール。

### 1.1 背景・課題

- 1年という時間は体感よりも早く過ぎる
- カレンダーは「日付」を伝えるが「年の中の位置」は伝えない
- 「あと何ヶ月」は計算が必要だが、針の位置なら一目で分かる

### 1.2 命名の由来

- Gazer = じっと見つめるもの — 1年をじっと観察する計器
- ドイツ語 Zeiger（時計の針）との音の類似 — Braun/Junghansへのリスペクト
- Stargazer（天体観測者）の連想 — 地球の公転=1年という天文的コンセプト

### 1.3 プロダクトの位置づけ

- 単品ウェブサイトとしてGitHub Pagesでリリース（v1）
- 将来的にSPCC（Side Panel Clock and Calendar）Chrome拡張機能の差別化要素として統合を検討
- WALL-CLOCKで培ったアナログUI表現の発展形
- アナログメーター/計器UIの技術的実験台

## 2. テーマシステム

### 2.1 設計思想

ANNUAL-GAZERのテーマは「色違い」ではなく「世界観の切替」。
テーマが変わると、バゲットカットの質感、フォント、背景、色調が一体として変化する。

### 2.2 テーマ一覧

```
ANNUAL-GAZER
├── INSTRUMENT（デフォルト）← v1 で完成させる
│   金属・計器・モノトーン・7セグ・インゴット
│
├── JEWEL（誕生石）← v2
│   宝石・ゴールド・月ごとの誕生石カラー・セリフ体
│
└── ZODIAC（星座）← v2
    天文・星空・星座記号・線画・光芒
```

### 2.3 INSTRUMENT テーマ（v1 スコープ）

v1で実装する唯一のテーマ。プロフェッショナルな計測器の世界観。

**キーワード:** 金属、インゴット、刻印、計器、モノトーン、7セグメント
**色調:** モノクロトーン基調、差し色（アクセントカラー）は1色のみ
**質感:** 金属の切削加工、旋盤仕上げ、インゴットの鋳造感
**フォント:** 7セグメントフォント（デジタル表示部）、細めのサンセリフ（その他）

### 2.4 JEWEL テーマ（v2 検討）

誕生石の色と質感をバゲットカットに反映。

**キーワード:** 宝石、ゴールド、誕生石、カラフル、ラグジュアリー
**変化する要素:**
- バゲットカットのセル: 表示中の月の誕生石の色・質感になる
  （1月=ガーネット/深紅、2月=アメジスト/紫、3月=アクアマリン/水色...）
- 内側のアナログ計器: ゴールドトーンに
- 中央テキスト: セリフ体に変更
- 背景: わずかに誕生石の色味を帯びる

### 2.5 ZODIAC テーマ（v2 検討）

十二星座と天文観測器の世界観。

**キーワード:** 天文、星空、星座、線画、光芒
**変化する要素:**
- 月ラベル: 数字/月名 → 星座記号に変更
- 背景: 星座の線画がうっすら浮かぶ
- バゲットカット: 星座に対応した色味
- 針: 星の光のようにわずかな光芒を持つ

## 3. 基本設計

### 3.1 全体構造: 二重円 + スペクトラムバー

```
                      ┌─ バゲットカットセグメント（31セル）
          外側の円 ───┤  インゴット風立体感、刻印数字
                      └─ 1ヶ月スケール

                      ┌─ 放射状の細かいバー群
          隙間 ───────┤  スペクトラムアナライザー風
                      └─ 24時間スケール + 末尾に時刻テキスト

                      ┌─ 内向きの目盛り（12ヶ月、365日分割）
          内側の円 ───┤  月ラベル（数字: 1, 2, 3...）内側に配置

          中央 ──────── 7セグメント風デジタル表示
                        年・月日・曜日・時刻・経過率・残日数
```

### 3.2 文字盤の分割方式

**日ベース均等分割を採用する。**

- 365日（うるう年は366日）で360°を等分する
- 1日 = 約0.986°（365日の場合）
- 月のラベル位置は12等分（30°間隔）ではなく、実際の日数に基づく

各月の開始角度（平年・1月始まり）:

| 1月 | 1 | 0.0° |
| 2月 | 32 | 30.6° |
| 3月 | 60 | 58.2° |
| 4月 | 91 | 88.8° |
| 5月 | 121 | 118.4° |
| 6月 | 152 | 148.9° |
| 7月 | 182 | 178.5° |
| 8月 | 213 | 209.1° |
| 9月 | 244 | 239.7° |
| 10月 | 274 | 269.3° |
| 11月 | 305 | 299.8° |
| 12月 | 335 | 329.4° |

## 4. 外側の円: バゲットカットセグメント

### 4.1 形状: バゲットカット

**形状の由来:** 高級時計のバゲットインデックス（Rolex Daytonaのアワーマーカー）。
長方形の四隅を45°で面取りした縦長の八角形。

- 各セルは縦長（放射方向に長い）の八角形
- 四隅を45°でカット（chamfer）— 角丸ではなく直線的なカット
- カットの大きさは控えめ（角を落とす程度）
- セルは円弧に沿って放射状に配置

### 4.2 質感: インゴット / 金属の塊

**INSTRUMENT テーマでの表現:**
- 面取りの辺ごとにわずかに明度を変えて立体感を出す
  - 上辺・左辺: わずかに明るい
  - 下辺・右辺: わずかに暗い
- ドロップシャドウは禁止。strokeの明暗差のみで表現
- 差はごく控えめに。「金属の塊に光が当たっている」程度

### 4.3 刻印数字

- 各セルの中に日数（1〜31）を表示
- 「彫り込んだような」「刻印された」表現:
  - 数字の色はセルのfillよりわずかに明るいか暗い（浮くのではなく沈んでいる）
  - 太字禁止。細めのフォントウェイト
  - letter-spacingを少し広めに取り、窮屈にしない

### 4.4 セルの配置

- MAX 31セル。12時方向 = 1日、時計回りに配置
- 月の日数に応じてセル数が変わる（28〜31個）
- 31日未満の月は1周させない（弧の途中でセルが終わる）
- セル間ギャップは均一

### 4.5 セルの3状態

**モノクロトーンを厳守する。オレンジ・緑など色分けは禁止。**

| 状態 | fill | stroke | 数字 |
|---|---|---|---|
| 消灯（未来の日） | なし or 極薄 | 背景よりわずかに明るい (opacity: 0.1〜0.15) | 目を凝らせば読める (opacity: 0.15) |
| 点灯（過ぎた日） | 暗めのグレー (rgba白, 0.2) | 点灯に合わせた明度 | 読める明度 (rgba白, 0.5) |
| 今日 | やや明るいグレー (rgba白, 0.3) | 明るく際立つ | 白に近い明るさ。最も際立つ |

### 4.6 土日の表現

- **セルのfill/strokeは平日と同じモノクロトーン**（セル自体に色は付けない）
- **数字の色だけ** で区別する:
  - 土曜: 数字をアクセントカラーで表示
  - 日曜: 数字をアクセントカラーの明るいバリエーションで表示

### 4.7 「今日」マーカー

- 今日のセルが最も目立つ（4.5参照）
- セル全体が「発光している」ような印象

## 5. 隙間: スペクトラムアナライザー風バー

### 5.1 バーの構成

- 二重円の隙間に放射状の細いラインが多数並ぶ
- 各バーは内円側から外円側に向かって伸びる短い放射状の線
- 本数はプロトタイプで調整（目安: 48本=30分刻み、または96本=15分刻み）
- 12時方向を起点に時計回り

### 5.2 バーの状態

| 状態 | 見え方 |
|---|---|
| 消灯（未来の時間） | 非表示、または極めて薄い (opacity: 0.03) |
| 点灯（過ぎた時間） | アクセントカラー、控えめな不透明度 |

### 5.3 グラデーション（動きの演出）

- 点灯済みバーは全部同じ明るさではなく、
  **末尾（現在時刻）に近いほどわずかに明るく、
  古いバーはわずかに暗くなる** グラデーションにする
- 10分更新のタイミングでグラデーションがシフト
- 「時間が流れている方向」が視覚的に分かる効果

### 5.4 時刻テキスト（末尾に表示）

- スペクトラムバーの点灯済み部分の末尾（=現在時刻の位置）に
  HH:MM:SS のテキストを表示
- **バーとテキストは重ならない**。テキストはバーの延長線上に置く
- テキストは **円弧に沿って湾曲** させる（SVG textPath を使用）
- 十分に大きいサイズで視認性を確保する

**文字の向き自動切替:**
- 円の右半分（12時〜6時方向）: 文字が外側を向く（通常の読み方向）
- 円の左半分（6時〜12時方向）: 文字が内側を向く（180°反転）

**12時方向付近の重なり対策:**
- テキスト位置が12時方向（バーの開始地点）に近づいた場合、
  テキストの背後に背景色の座布団（角丸矩形）を敷く

### 5.5 24時間モード（v1 デフォルト）

- 24時間で1周
- 0時 = 12時方向から開始

### 5.6 12時間モード（v2 で実装）

- 12時間で1周、AM/PMで2周
- AM/PMで色を変える
- v1ではUI上は disabled で表示

## 6. 内側の円: アナログ計器

### 6.1 目盛り

- 内向き（円の内側に向かって伸びる）
- 日ベース均等分割（365日で360°を等分）

**目盛りの階層:**

| 階層 | 対象 | 太さ | 長さ |
|---|---|---|---|
| レベル1 | 月の区切り（12本） | 太い | 長い |
| レベル2 | 週区切りまたは10日区切り | 中 | 中 |
| レベル3 | その他の日 | 細い | 短い |

### 6.2 月ラベル

- **デフォルトは数字表示: 1, 2, 3, 4... 12**
- JAN/FEB 表示は設定で切替可能（v2検討）
- ラベルは内側に配置

### 6.3 年進捗針（唯一の針）

- 年間通日ベースで連続的に動く
- 1月1日 = 0°（12時方向）、12月31日 ≈ 359°
- 1周 = 1年
- 細く繊細なデザイン（計器の針の印象）

## 7. 中央エリア: 7セグメント風デジタル表示

### 7.1 表示する情報

```
        2026
       03.31
        TUE
       17:27
      ─────────
       24.7%
      275 days left
```

**上段（日付・時刻ブロック）:**
- 年: 2026
- 月日: 03.31
- 曜日: TUE（またはMON, WED...）
- 時刻: 17:27（またはHH:MM:SS、設定に準拠）

**下段（経過情報ブロック）:**
- 年間経過率: 24.7%
- 今年の残り日数: 275 days left

上段と下段の間に薄いセパレータ（線または余白）を置く。

### 7.2 7セグメントフォント

- **数字は7セグメント（デジタル計器）フォントで表示する**
- 対象: 年（2026）、月日（03.31）、時刻（17:27）、経過率（24.7%）、残日数（275）
- 曜日（TUE）、単位（days left、%）は通常のサンセリフフォント

**7セグの表現:**
- Webフォントを使用（例: DSEG7 Classic 等の7セグフォント）
- 消灯セグメントをごく薄く表示すると、液晶のゴーストが見えるリアリティが出る
  （例: 全セグメントを opacity: 0.05 で表示、点灯セグメントを opacity: 0.8 で上書き）
- 外側のG-Shockセグメントと呼応して「デジタル計器」の世界観を強化

### 7.3 時刻表示の設定

| 設定値 | 表示 |
|---|---|
| HH:MM:SS | 秒まで表示（秒は常時更新） |
| HH:MM | 秒なし |
| HH:MM + blink colon | 秒なし、「:」が1秒おきに点滅 |

- 中央の時刻が秒まで表示される場合、ここが「動いている」ことの主な視覚的サインになる

### 7.4 情報の優先度とサイズ

- 時刻と月日を最も大きく（これが一番頻繁に見る情報）
- 年は中サイズ
- 経過率と残日数は小さめ（補助情報）
- 全体が窮屈にならないよう、情報間の余白を十分に取る

## 8. 背景

### 8.1 INSTRUMENT テーマの背景

真っ黒の単色ではなく、控えめな質感を加える。

**方針:** 「計器の筐体の中」または「宇宙空間に浮かぶ計器」

以下のいずれか、またはプロトタイプで比較して採用:

**案A: サンレイ仕上げ（金属テクスチャ）**
- 文字盤の背後にごく薄い同心円のテクスチャ
- 高級時計の文字盤にあるサンレイ仕上げ（旋盤加工痕）の表現
- 「計器の筐体の内側」としての物質感
- opacity: 0.02〜0.05程度、目を凝らさないと気づかないレベル

**案B: 微細な光点（スターフィールド）**
- 極めて控えめな星のような光点を背景に散らす
- Stargazer（天体観測者）のコンセプトと呼応
- 光点は静止、または非常にゆっくり明滅するものが数個
- 「宇宙の中に浮かぶ計器」という世界観
- やりすぎ禁止。プラネタリウムアプリにしない。

### 8.2 背景のNGライン

- 背景は主張しない。計器本体より目立つことは絶対にない
- アニメーション過多は禁止
- 背景だけ見ても何があるか分からないぐらいが正解

## 9. アニメーション: 「生きている」感の演出

### 9.1 呼吸する光（ピボットポイント）

- 年進捗針の中心点（ピボット）がごくゆっくり明滅する
- 周期: 4〜6秒（呼吸のリズム）
- 振幅: opacity 0.3〜0.6 程度の範囲
- ぱっと見では気づかないが、じっと見ると「生きている」と分かる
- 「この計器は稼働中である」というサイン
- CSS animation の ease-in-out で自然な呼吸感を出す

### 9.2 スペクトラムバーのグラデーション

- セクション5.3 参照
- 末尾に近いバーほど明るく、古いバーほど暗い
- 10分更新でグラデーションがシフト

### 9.3 中央時刻の更新

- HH:MM:SS 表示の場合、秒が1秒ごとに更新される
- blink colon の場合、「:」が1秒おきに点滅
- これが計器全体の「動いている」ことの最も分かりやすいサイン

### 9.4 10分更新（Pixel Shifter 方式）

- 針、スペクトラムバー、中央テキストの経過情報を10分ごとに再計算
- CSS transition 0.3〜0.5秒でスムーズに遷移
- WALL-CLOCKのPixel Shifter 2.0 / Zero-Waste Engine の思想を継承

### 9.5 アニメーションのNGライン

- 派手な動き、点滅、パルス、バウンス等は禁止
- すべてのアニメーションは「気づく人だけ気づく」レベルに留める
- 計器としての品位を損なわない

## 11. デバッグモード

### 11.1 概要

任意の日付・時刻を指定して表示を確認できるモード。
デザイン確認やエッジケースの検証に使用する。

### 11.2 UI

- 設定パネルの DEBUG セクション内に配置
- Debug Mode トグル（ON/OFF）
- ONにすると展開:
  - 日付ピッカー（yyyy-mm-dd）
  - 時刻入力フィールド（HH:MM、空欄可）

### 11.3 モード1: 日付のみ指定（時刻フィールドが空）

指定日に変わる要素:
- 外側バゲットカット: 指定日の月の日数でセル生成、指定日まで点灯
- 内側年進捗針: 指定日の角度に移動
- 中央テキスト: 指定日の日付・曜日・年間経過率・残日数

リアルタイムのまま:
- スペクトラムバー + 時刻テキスト
- 中央の時刻表示

### 11.4 モード2: 日付+時刻を固定（時刻フィールドに値あり）

全要素が指定日時で固定:
- スペクトラムバー: 指定時刻まで点灯して停止
- 中央の時刻: 指定時刻を表示（秒は00固定）
- アニメーション: 呼吸する光を含め全停止
- スクリーンショット撮影やデザインレビュー用

### 11.5 DEBUGバッジ

- デバッグモードON時、画面右上に小さく「DEBUG」バッジ表示
- 控えめなスタイル（opacity: 0.4程度）

### 11.6 状態の保持

- LocalStorageに保存しない。リロードで常に通常モードに戻る

## 12. デザイン

### 12.1 INSTRUMENT テーマのデザインポリシー

3つの質感 + ガジェット感を融合する。

**構造 — 無印良品 アナログ壁掛け時計**
- 情報の階層を色ではなく太さ・長さで表現
- 装飾を排除する。「読ませない」デザイン

**精密感 — アナログVUメーター / McIntosh**
- 針は細く繊細に、目盛り線はシャープに
- 「時を測定する計器」という静かな緊張感

**空間・コントラスト — Bang & Olufsen**
- ダーク背景に高コントラストの前景
- 要素間の余白を十分に確保
- 補助情報は主情報より明度を落とし前後感を出す

**ガジェット感 — KORG NTS-2 / Casio G-Shock**
- 外側のバゲットカットセグメントはインゴット/金属塊の印象
- スペクトラムバーはオーディオ機器のアナライザー
- 7セグメント表示はデジタル計器の血統
- デジタルとアナログの共存が生む計器美学

**ラグジュアリー — Rolex Daytona バゲットインデックス**
- バゲットカットの形状は高級時計のインデックスから
- 精密にカットされた金属の塊としての存在感
- 刻印数字の控えめな美しさ

### 12.2 カラー基本方針

- ダークテーマのみ（v1）。デザイン判断はすべてダーク基準
- 色数は3色以内: 背景・前景・アクセント
- バゲットカットセグメントはモノクロトーンを厳守
- アクセントカラーの用途: スペクトラムバー点灯、土日の数字色、今日マーカー

### 12.3 レイヤー間の余白

- 各レイヤー間の間隔を十分に確保
- 詰まった印象になるぐらいなら各要素を細くして余白を確保
- **余白がデザインの品質を決める**

### 12.4 NGリスト

- **フォント:** 丸ゴシック・カジュアルフォント禁止
- **エフェクト:** ドロップシャドウ・グロー・グラデーション禁止（立体感はstroke明暗差のみ）
- **装飾:** ベゼル・外枠・装飾線禁止
- **余白:** レイヤー間の間隔が詰まっている状態は禁止
- **針:** 太すぎる針は禁止（文字盤半径の1%以下）
- **セグメント:** セルが大きすぎる・角丸が強すぎるのは禁止。バゲットカットは直線的カット
- **バー:** スペクトラムバーが太すぎる・派手すぎるのは禁止
- **色:** 3色を超える配色禁止
- **バゲット色:** モノクロトーン以外の色をセルに付けることは禁止（土日は数字色のみ）
- **アニメーション:** 派手な動き・点滅・パルス・バウンス禁止
- **背景:** 背景が計器本体より目立つことは禁止
- **全般:** 「足す」ことで解決しようとする発想を禁止

## 15. 技術方針

### 15.1 描画

- SVGベースを推奨
- バゲットカット: `<polygon>` または `<path>` でchamfered rectangle
- スペクトラムバー: `<line>` または `<rect>` を放射状配置
- 時刻テキスト: `<textPath>` で円弧に沿わせる
- 背景テクスチャ: SVGパターンまたはCSS
- アニメーション: CSS transition（10分更新）、CSS animation（呼吸する光）

### 15.2 フォント

- 7セグメント: DSEG7 Classic 等のWebフォント
- その他: 細めのサンセリフ（WALL-CLOCKと統一）

### 15.3 フレームワーク

- WALL-CLOCKの構成を踏襲: React + TypeScript + Vite
- 状態管理: React hooks + LocalStorage

### 15.4 配信

- GitHub Pages（v1）
- Chrome Web Store / SPCC統合（将来）
